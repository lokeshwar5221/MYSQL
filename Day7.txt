TCL(Transaction Control Language)
--------------------------------
def
---
The process of combining all related operations into a single unit and executing them according to 
the rule "either all operations should be succeed or none should be succeed".is called Transaction
Management.

A transaction is considered a single unit of work and fallows the principal

"either all or none"

example: Fund transfer

consider a money transfer process between two accounts.

1.Debit amount from the sender account
2.Credit amount into receivers account.

both Operations should be executed as a single transaction

if the amount is debited from the sender account successfully. But crediting the recovers 
account fails.it may lead to the data inconsistency problems.

There are two types of transactions such as 
1.Local Transactions
--------------------
in local transactions, all operations of a transaction are executed in the same database.

Example
-------
funds transfer from one account to another account where both accounts belong to the same bank


2.Global Transactions.
----------------------
In global transactions, operations of a transaction are executed on different databases.

Example
-------
Fund transfer from one account to another account where both accounts belong to the different bank.

ACID property
-------------
Automictic:-all or nothing.
Consistency:-The database remains in valid state
isolation:-Transaction do not interface with each other
Durability:-committed changes are permanent

WHAT IS LOCKING
===============
 locking is a mechanism used by the database to control concurrent access data.

it ensures that multiple users access the same data simuntaniously the database maintain
consistanct ,integrity, and prevent conflicts.

1.shared lock
--------------
definition

a shared lock allows a transaction to read data 
multiple transactions can hold shared lock on the same row.
no transaction can modify the row until all shared locks are released.

#Characteristics#

read only
multiple users can read simontaniously
block updates and deletes

2.Exclusive lock
----------------
definition

An exclusive lockallows an transaction to modify data 
while an exclusive lock exists:
no other transactions can read using a locking read or modify that row.
only the transaction holding the lock can update/delete the data.

#Characterstics#

used for INSERT,UPDATE,DELETE
only one exclusive lock at a time
Block both reads(locking reads) and writes

3.Intent lock
-------------
An intent lock is a table level lock that indicates a transaction intends to place row-level
locks.
It helps MySQL quickly determine whether a table-level lock conflicts  with existing row locks.
Users do not usually create insert locks manually.in Db manages them automatically.

Real-life Example 
Imagine a library :
Shared lock->Many students can read the same book at the same time.
Exclusive Lock-->one student borroe the book,so nobody else can read or borrow it until it is returned
Intent Lock-->The librarian marks the someone is about to read or borrow books from a shelf,
helping coordinate access without checking every book individualy.
